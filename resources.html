Resources & Data

Open datasets and data-processing code.

Open datasets

Datasets used or referenced in this project for menstrual cycle and ovulation inference.

Dataset Source / link Scale Notes
mcPHASES PhysioNet (DOI: 10.13026/zx6a-2c81) 42 participants; two study periods Wearable physiological signals, daily hormone assays (e.g. LH), self-reported menstrual health. Used for physiology-informed, day-level modelling.
Menstrual Cycle Data (Kaggle) Kaggle Self-reported cycle records Used for multi-modal self-reported baseline reproduction (e.g. DT, RF, LSTM on cycle-structure features).

Data-processing code

Pipeline for hormonal data alignment, ovulation day annotation (LH surge onset/peak + 1 day, multi-peak handling), and construction of temporally aligned cycle-level and day-level representations. Self-reported data: subject-specific mean imputation, cycle-level structural features. Code and usage instructions will be linked here (e.g. GitHub repository).

Usage notes

mcPHASES: check PhysioNet for citation and license. For ovulation annotation, hormonal measurements are used only as labels, not as model inputs, to simulate wearable-only deployment. Subject-level or cycle-level splitting is recommended to avoid leakage. Kaggle data: see dataset page for terms of use.
